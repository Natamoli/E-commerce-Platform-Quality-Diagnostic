# E-commerce-Platform-Quality-Diagnostic

## 📋 Project Overview
This project represents a comprehensive QA assessment conducted for a leading D2C E-commerce platform. The objective was to evaluate the website's functionality, usability, and UI consistency across key user flows (e.g., cart management, checkout, navigation).

## 🚀 Key Metrics
* **Testing Scope:** Full Website Exploratory Testing.
* **Bugs Identified:** 15
* **Test Focus:** Functional, UI/UX, and Cross-Platform consistency.
* **Tools Used:** Google Chrome DevTools, Manual Regression Testing, Spreadsheet Documentation.

## 🛠️ Testing Methodology
My approach focused on the **End-to-End (E2E) User Journey**. I treated the website not just as a collection of features, but as a critical path for user conversion:

1.  **Exploratory Testing:** Navigated the platform as a first-time user to identify friction points and broken flows.
2.  **UI/UX Validation:** Checked for visual inconsistencies, responsive design issues, and accessibility errors.
3.  **Severity & Priority Analysis:** Each bug was classified by its impact on the business (Revenue/Conversion) vs. the user experience.

## 📊 Summary of Findings
During this audit, I identified **15** bugs. The findings were documented with the following criteria for the development team:
* **Severity:** Critical, Major, Minor, Low.
* **Steps to Reproduce:** Clear, concise instructions to ensure the dev team can replicate the issue.
* **Expected vs. Actual Results:** Highlighting the discrepancy.
* **Evidence:** [Attached screenshots/video clips for visual verification.](https://docs.google.com/spreadsheets/d/1UnhhVuf5Nij5qZRTBDOvqpVYvrb7XBmP/edit?usp=sharing&ouid=110641800876252641675&rtpof=true&sd=true)

## 🎓 Key Learnings (The "QA Mindset")
Performing this audit allowed me to sharpen several core QA competencies:

* **Business Impact Assessment:** I learned to prioritize bugs based on how they affect the "Golden Path" (the journey from landing page to successful purchase). A visual glitch is annoying, but a checkout error is a business emergency.
* **Precision in Documentation:** I improved my ability to write clear, reproducible bug reports that bridge the gap between "what I saw" and "what the developer needs to fix it."
* **The "Customer-Centric" Lens:** By applying my background in customer service, I was able to identify "low-hanging fruit" improvements that would reduce support tickets and improve the overall user experience.
